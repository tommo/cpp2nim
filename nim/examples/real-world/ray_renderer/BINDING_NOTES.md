# Ray Renderer Nim Bindings

Nim bindings for [Ray](https://github.com/sergcpp/Ray) - a small physically-based renderer.

## Generation

```bash
cd examples/real-world/ray_renderer
bash test_pipeline.sh  # or manually:
../../../bin/cpp2nim_cli all --config=config.json headers/Types.h headers/Log.h headers/SceneBase.h headers/RendererBase.h
```

## Configuration Strategy

### Public API Extraction
Only public headers are processed. Internal headers are excluded via `ignore_files`:
- `internal/*` - Implementation details
- `VulkanFunctions.h`, `Config.h`, `Bitmask.h`, `Span.h` - Platform/utility headers

### Type Handling

**Ignored types** (platform-specific, automatically filtered by generator):
- VulkanDevice, VulkanFunctions, VkImage, VkImageView, VkCommandBuffer
- ID3D12Resource, ID3D12DescriptorHeap, ID3D12GraphicsCommandList
- Bitmask, Span, remove_all_const

The generator's `shouldIgnoreType()` function:
- Strips pointer/reference/const qualifiers
- Strips namespace prefixes (e.g., `Ray::VulkanDevice` → `VulkanDevice`)
- Extracts base type before template params
- Excludes convertible types (Span→pointer, Bitmask→uint32, etc.)

**Inheritable types** (use `byref` pragma):
- ILog, RendererBase, SceneBase

### Post-Processing Fixes

The `post_fixes` config section applies regex/plain text replacements after generation.
Only 4 rules are needed (reduced from 8+ through generator improvements):

1. **shared_types.nim**: Adds Handle types (CameraHandle, LightHandle, etc.) generated by C++ DEFINE_HANDLE macro that libclang can't parse

2. **Types.nim**: Fixes color_t/color_data_t template structs and adds type aliases

3. **RendererBase.nim**:
   - Fixes std::pair return type to pointer
   - Fixes array template param order

## Known Limitations

1. **DEFINE_HANDLE macro**: C++ macro-generated types must be manually added via post_fixes
2. **std::pair return types**: Simplified to `pointer`
3. **std::function parameters**: Simplified to `pointer`

## Example Usage

See `example.nim` for a complete usage example that demonstrates:
- Creating renderer with settings
- Setting up a camera
- Adding lights and materials
- Creating mesh geometry
- Rendering and denoising

```nim
import output/[shared_types, Types, Log, SceneBase, RendererBase]

# Create renderer
var settings: settings_t
settings.w = 640
settings.h = 480
let renderer = createRenderer(settings, addr g_stdout_log)

# Create scene and add camera
let scene = renderer.createScene()
var cam: camera_desc_t
cam.origin = [0.0'f32, 0.0, 5.0]
cam.fwd = [0.0'f32, 0.0, -1.0]
cam.fov = 45.0
let camHandle = scene.addCamera(cam)

# Render
var region = newRegionContext(rect_t(x: 0, y: 0, w: 640, h: 480))
for i in 0..<16:
  renderer.renderScene(scene[], region)
  region.iteration.inc
```

## Generator Improvements Made

These fixes were added to the cpp2nim generator to reduce post_fixes needs:

1. **Rvalue reference handling**: C++ `&&` refs treated as pass-by-value (not `var`)
2. **Ignore type filtering**: `shouldIgnoreType()` properly handles:
   - Namespace prefixes (`Ray::VulkanDevice`)
   - Pointer/const qualifiers (`ID3D12Resource*const`)
   - Distinguishes skippable types from convertible types (Span→pointer)
3. **Constructor/method filtering**: Automatically skips functions with ignored type params
4. **Field filtering**: Skips struct fields with GPU-specific types
5. **Template param cleanup**: Filters empty template params to avoid trailing commas
6. **Span/pair/function handling**: Early detection in `getNimType` before template conversion
7. **stdint.h types**: Added uint8_t, uint32_t, etc. to BasicTypeMap
8. **Underscore identifiers**: Enum values starting with `_` prefixed with `x`
